# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a persisted light/dark theme toggle to Morse Walker using Bootstrap 5.3's native `data-bs-theme` attribute.

**Architecture:** An inline `<head>` script resolves the theme on first paint (stored preference, else OS `prefers-color-scheme`) and sets `data-bs-theme` on `<html>`. A header icon button toggles the theme; the choice is persisted to `localStorage`. A `setTheme()` helper in `app.js` keeps the attribute, button icon, and `theme-color` meta tags in sync.

**Tech Stack:** Plain HTML/CSS/JS, Bootstrap 5.3 (Bootswatch cerulean), Font Awesome 6 (JS/SVG build), Webpack 5, `localStorage`.

## Global Constraints

- No new dependencies. Use only what is already imported (Bootstrap 5.3, Font Awesome 6).
- No build/webpack config changes.
- `localStorage` key for the theme is exactly `theme`; values are exactly `'dark'` or `'light'`.
- Dark `theme-color` is `#212529` (Bootstrap dark body bg); light stays `#fafafa`.
- Commits must use `--no-gpg-sign` (repo requires signing but no key is configured in this environment).
- The husky pre-commit hook runs Prettier on `src/**/*.{js,css,html}`; written code should already match Prettier formatting to avoid reformatting churn.

### Task 1: First-paint theme resolution (no flash)

**Files:**
- Modify: `src/index.html` (insert inline script just before `</head>`, after the existing `theme-color` meta tags at lines 74-76)

**Interfaces:**
- Consumes: nothing.
- Produces: on page load, `document.documentElement` has attribute `data-bs-theme` set to `'dark'` or `'light'`; `<meta name="theme-color">` content matches. The toggle in Task 2 reads `data-bs-theme` to determine current state.

- [ ] **Step 1: Add the inline resolution script**

In `src/index.html`, immediately before the closing `</head>` tag (the line after the `msapplication-TileColor` meta), insert:

```html
    <!-- Resolve theme before first paint to avoid a flash of the wrong theme -->
    <script>
      (function () {
        var stored = localStorage.getItem('theme');
        var theme =
          stored ||
          (window.matchMedia &&
          window.matchMedia('(prefers-color-scheme: dark)').matches
            ? 'dark'
            : 'light');
        document.documentElement.setAttribute('data-bs-theme', theme);
        var color = theme === 'dark' ? '#212529' : '#fafafa';
        var metas = document.querySelectorAll('meta[name="theme-color"]');
        for (var i = 0; i < metas.length; i++) {
          metas[i].setAttribute('content', color);
        }
      })();
    </script>
```

- [ ] **Step 2: Manually verify OS-driven default**

Run: `npm start` and open the app in a browser.
- With OS/browser set to **dark** and no `theme` in `localStorage` (run `localStorage.removeItem('theme')` in devtools, then reload): the page renders dark. Confirm `document.documentElement.getAttribute('data-bs-theme')` is `'dark'` in the console.
- With OS set to **light** and no stored preference: renders light, attribute is `'light'`.
Expected: the page background and Bootstrap components match the OS preference; no visible flash from light→dark on load.

- [ ] **Step 2: Add the `setTheme` helper to `app.js`**

In `src/js/app.js`, add this module-level function. Place it just before the `document.addEventListener('DOMContentLoaded', ...)` block (i.e., immediately before line 72, after the `farnsworthLowerBy` declaration and its doc comment):

```javascript
/**
 * Applies a theme ('dark' or 'light') across the UI.
 *
 * - Sets the `data-bs-theme` attribute Bootstrap uses to recolor components.
 * - Swaps the toggle button icon (moon while light, sun while dark) and aria-label.
 * - Keeps the browser `theme-color` meta tags in sync with the active theme.
 */
function setTheme(theme) {
  document.documentElement.setAttribute('data-bs-theme', theme);

  const toggle = document.getElementById('themeToggle');
  if (toggle) {
    toggle.innerHTML =
      theme === 'dark'
        ? '<i class="fa-solid fa-sun"></i>'
        : '<i class="fa-solid fa-moon"></i>';
    toggle.setAttribute(
      'aria-label',
      theme === 'dark' ? 'Switch to light mode' : 'Switch to dark mode'
    );
  }

  const color = theme === 'dark' ? '#212529' : '#fafafa';
  document
    .querySelectorAll(
      'meta[name="theme-color"], meta[name="msapplication-TileColor"]'
    )
    .forEach((meta) => meta.setAttribute('content', color));
}
```

- [ ] **Step 3: Wire init + click handler in `DOMContentLoaded`**

In `src/js/app.js`, inside the `DOMContentLoaded` callback, add the theme toggle wiring. Add the element lookup alongside the other `document.getElementById` lookups near the top of the block (after `const stopButton = ...` on line 78), and the init/listener alongside the other event listeners (after `stopButton.addEventListener('click', stop);` on line 94):

Element lookup (with the other lookups):

```javascript

```

Init + listener (with the other event listeners):

```javascript
  // Theme toggle: sync icon/meta to the theme resolved in <head>, then toggle on click
  setTheme(document.documentElement.getAttribute('data-bs-theme') || 'light');
  themeToggle.addEventListener('click', () => {
    const next =
      document.documentElement.getAttribute('data-bs-theme') === 'dark'
        ? 'light'
        : 'dark';
    setTheme(next);
    localStorage.setItem('theme', next);
  });
```

- No flash of the wrong theme on reload.
Expected: all of the above hold.
